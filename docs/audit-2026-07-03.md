**Date:** 2026-07-03 · **Read-only audit — no code changes.**
**Scope:** shopos-core 1.21.24, shopos-digital 1.7.3, shopos-theme 1.11.26 (`vendor/`, `node_modules/`, `dist/` excluded).
**Method:** four parallel specialist passes (security, performance, hardcoded values, loose ends); every high-impact single-source claim was independently re-verified against the code before inclusion. ## 1. Security

**Posture: strong.** No SQL injection, no missing nonce/capability checks, no privilege escalation, no SSRF, no unsafe `unserialize`, no open redirects, no credentials in the repo. The highest-risk surfaces (ShopFilters/Search dynamic SQL builders) are exemplary — fully placeholdered, numerics cast, table names as constants, alias sanitisation. All 15 shopos-digital AJAX handlers pair `check_ajax_referer` + `manage_options`. Public `nopriv` endpoints (QuickView/Search/ShopFilters) are read-only, nonce-gated, and rate-limited.

### Findings

| Sev | Location | Issue |
|-----|----------|-------|
| **Medium** | `shopos-core/src/Modules/RestockNotify/CSV_Exporter.php:112-127` + `legacy/includes/class-shopos-restock-admin.php:271` | CSV formula injection (Top-10 #2). Fix: prefix fields starting with `= + - @ \t \r` with `'` before `fputcsv`, both exporters. Legacy twin falls under the Hard Rule #3 migration plan. |
| **Medium** (hardening) | `shopos-digital/includes/class-shopos-digital-indexes.php:416` (+ ALTER path :231) | `CREATE INDEX` interpolates index name/columns into DDL unprepared. Values come from hardcoded definition maps (selection is `in_array`-validated), so **safe today** — but one refactor away from admin-level SQLi and DDL can't be parameterized. Mirror the table-name allowlist check `ajax_convert_myisam` already does (:1003-1008). |
| Low | `shopos-digital/includes/class-shopos-digital-admin.php:717` | Profiler `recommendation` echoed unescaped (every sibling field on the row uses `esc_html`). Internally generated today; wrap for consistency. |
| Low | `class-shopos-digital-admin.php:710, 714` | `$type_badge` / `$color` echoed unescaped; effectively enum constants — `esc_attr` for defense-in-depth. |
| Low | `Core/Security.php:93-102` | Rate limiter keys on `REMOTE_ADDR` per-IP transients — shared bucket behind NAT/CDN, and distributed clients each get their own budget. Acceptable now; consider a global ceiling if abused. |

### Verified clean (highlights)
- ShopFilters `Index_Repository`/`Query`/`Query_Builder`: `%d`-placeholder `IN()` lists, `(float)`/`%F` price bands, `preg_replace` alias sanitisation, slugs via `get_term_by` not SQL.
- Search `Query_Engine`/`Search_Repository`: fully prepared MATCH…AGAINST, `esc_like` on LIKE args, int-cast LIMIT capped at 500.
- Legacy RestockNotify: orderby whitelist, prepared queries, CSPRNG unsubscribe token (`random_bytes(16)`), nonce+honeypot+rate-limit on subscribe, header-injection-stripped email From.
- ProductFeed: all XML through `ENT_XML1|ENT_QUOTES`, fixed uploads path, no remote fetches anywhere in the suite.
- Settings import: `is_uploaded_file` + JSON + version/prefix allowlist (`shopos_core_*`/`shopos_digital_*` only); all redirects `wp_safe_redirect` + `exit`.

## 2. Performance

**Posture: deliberate and mostly sound** — fact tables with correct indexes, debounced batched indexers, request memoization; the 1.21.19–1.21.24 fixes verifiably do what they claim. shopos-digital practices what it preaches (profiler time-boxed, caches gated + targeted invalidation). Remaining risk clusters in four places:

### A. ShopFilters read path / facet cache
- **A1 (High-Med)** — dead unbounded native search `WP_Query` per search pageview (Top-10 #5). `Query_Builder.php:671-697` + `Search/Results_Query.php:331-336`.
- **A2 (Med)** — unbounded, user-controlled cache-key cardinality (`Query_Builder.php:635-655`, `Url_State.php:44-63`): validate `filter_<tax>` params against indexed taxonomies/facet config before keying; don't `set_transient` junk states.
- **A3 (Low-Med)** — `paged` in the cache key though facets are page-invariant; `?paged=N` requests re-run the full postings sweep. Key on state-minus-paged.
- **A4 (Med)** — no stampede lock + `bump_rev()` on every order-driven stock write (`Indexer.php:170-182`): effective TTL ≈ inter-order gap, synchronized rebuilds. Add a short rebuild lock / serve-stale, debounce rev bumps.
- **A5 (Med)** — uncached facet build is N+1 on `get_term`/`get_term_meta`/attachment lookups (`Query_Builder.php:747-829`): ~2-4 queries per facet term without an object cache. Prime with one `get_terms(update_term_meta_cache=true)` per taxonomy.
- **A6 (Low)** — `product_prices()` + `product_flags()` are two identical-id-list `meta_lookup` sweeps; merge into one SELECT.
- **A7 (Low)** — `all_product_ids` (`in_stock=1`) can't use the secondary index; `index_has_data` uses `COUNT(*)` where `LIMIT 1` suffices; `instock_product_ids` not memoized (runs up to 3× per filtered request).
- **A8 (Low)** — invalidation misses: facet-config/price-bands/label/style saves and term renames don't `bump_rev()` (stale ≤ 5 min, acceptable but one line each). Uninstall doesn't delete `_transient_shopos_core_sf_q_*` / `fmrl_*` rows (TTL-bounded).

### B. Indexers & background work
- **B1 (Med)** — 2× `as_next_scheduled_action()` on `init` on every request (`ShopFilters/Module.php:146`, `Search/Module.php:140`). Gate to admin/cron or cache the fact.
- **B2 (Med)** — ShopFilters reindex calls `get_available_variations()` (full frontend payload, ~10+ queries/variation) just for stock flags (`Indexer.php:329`); use `'objects'` or read `meta_lookup` directly. Milder twin: Search `variation_skus()` (`Indexer.php:301-316`).
- **B3 (Low-Med)** — idle churn: both reconcile loops write LAST_RUN + WATERMARK options per 5-min tick ≈ 1,150 option writes + 576 WP_Queries/day at idle; Action Scheduler accrues ~17k completed rows at 30-day retention.
- **B4 (Low)** — `mark_dirty()` read-modify-writes the queue option per product event (bulk-import amplification, lost-update risk across workers).
- **B5 (Low-Med)** — search read is a deliberate full-table scan (leading-wildcard LIKE fallback); fine at current size, linear with catalog growth; `KEY in_stock` is a low-cardinality no-op.

### C–D. Widgets / legacy
- **C1 (High)** — ProductSlider meta-orderby (Top-10 #3).
- **D1 (Med-High, legacy)** — VariationSwatches archive picker keys per-product transients on WC's *group* `product` transient version (`legacy/includes/class-archive.php:266-274, 454, 548`): every product save orphans **all** cached grids → miss storms + `wp_options` bloat on a fast-moving store. Fix belongs to a Hard Rule #3 migration plan: per-product invalidation instead of group version.

### E. Misc
- `Settings_Hub.php:259` saves options without `autoload => false` (minor bloat).
- Theme cache-bust constant mismatch (Top-10 #1) — also independently found by the loose-ends pass.
- Verified OK: ShopFilters shortcode-scoped asset enqueue; Search site-wide-by-design; feed generation flock-guarded + batched; sampler's unbounded enumerations are event-driven not per-request; `SELECT *` on subscribers is admin-export-only.

**Coverage gap:** `class-shopos-digital-frontend/woocommerce/bloat/database/autoload` were verified at hook-wiring level, not line-audited; three Low "verify" caveats noted in shopos-digital (profiler `maybe_create_table` on `admin_init`, activity-log call sites, dynamic `get_terms` closure at `class-shopos-digital-admin-cache.php:138`).

## 3. Hardcoded values

**Credentials: clean** (no keys/secrets/emails/non-TLS URLs). Tables all `$wpdb->prefix`-built; AJAX URLs all localized; nonce actions distinct; currency via WC APIs.

### Store-specific (the productization blockers)
1. **Elementor Style Kit slot IDs in plugin CSS** — `sk_type_12`/`sk_type_2` referenced in `VariationSwatches/assets/css/shopos-*.css` (5 sites) and `shopos-theme/assets/css/shopos-tokens.css:63-64`. These are *this store's* kit IDs; plugin CSS should reference only `var(--fm-font-body, inherit)` and let the theme own the kit mapping.
2. **Hebrew/RTL leakage** (Top-10 #8): hardcoded RTL email shell (`Email.php:232-233`; legacy twin `class-shopos-restock-email.php:99`); Hebrew JS literals bypassing Labels (`shopos-swatches.js:219` — the keys exist in `Labels.php`; `RestockNotify frontend.js:72` no `t()` wrapper; lines 47-116 Hebrew *fallbacks*). `VariationSwatches/Labels.php` He/En binary with no filter/options is a documented owner decision — add a filter when productizing. Legacy Hebrew-msgid anti-pattern noted for the migration plan.
3. **`shopos-*` naming leak** — 22 CSS + 14 JS occurrences in InfiniteScroll, **plus ProductSlider CSS depending on `.shopos-skeleton`** (`product-slider.css:397-407`) — a rename now spans three files in two modules. Also `shopos_debug=1` query-param alias (`infinite-scroll.js:13`).
4. Cosmetic: "arba4" in comments (`Results_Query.php:200`, `Gallery_Slider.php:10`), `.elementor-kit-59` cite (`quick-view.css:27`); Hebrew font stacks in theme.json (fine as theme defaults).

### Duplicated constants (one-edit-away bugs)
- **768px mobile breakpoint in 5 files, 2 languages** (shop-filters.js/css, shopos-swatches.js/css, search.css) — no shared source; JS drawer behavior desyncs silently if CSS moves.
- **640/1024 card breakpoints mirrored PHP↔CSS** (`ProductSlider/Widget.php:996-999` ↔ `product-slider.css` + `category-slider.css`) — comment-enforced only; a CSS change silently re-breaks the 1.21.23 image-bytes fix.
- ShopFilters show-more `8` (JS) vs `:nth-child(n+9)` (CSS); drawer CSS `0.3s` vs JS `350ms` fallback; indexer trio duplicated across Search/ShopFilters (deliberate but unmarked); AJAX rate limit `30/60` repeated verbatim in 3 modules.
- **Z-index**: theme defines a token scale (`--fm-z-*`, shopos-tokens.css:169-174) but only Search uses it; VariationSwatches `9999!important`/`10000`, ShopFilters `99998/99999`, QuickView `100000` (exceeds the theme's `--fm-z-max`). Stacking order between drawers is an accident of literals.

### Tunable magic numbers worth a filter (additive, cheap)
- Facet-cache TTL `5*MINUTE_IN_SECONDS` inline (`Query_Builder.php:616`) — the knob a store owner most plausibly wants.
- `Search MAX_RESULTS=500`; indexer `BATCH/SWEEP/DEBOUNCE` constants (both modules); dropdown clamp bounds `1..20`; InfiniteScroll `rootMargin: 800px` (the most perf-relevant knob is the untunable one, while `maxPages`/`hybridThreshold` are settings); ShopFilters JS `DEBOUNCE_MS=400` (Search's equivalent *is* a setting — inconsistent).
- **Wrong source of truth**: per-page from `posts_per_page` (blog setting, fallback 12) in `Query_Builder.php:955` and — consequentially — `Query.php:297-298` where it *slices the actual grid*; should prefer `loop_shop_per_page`.
- ProductSlider archive thumb hardcoded `'large'` (`Widget.php:1231`) — scoped correctly, value just needs a filter.
- shopos-digital: `cdnjs.cloudflare.com` preconnect added unconditionally when `fe_add_preconnect` is on (`class-shopos-digital-frontend.php:158`) — useless connection for sites not using cdnjs; make it a toggle/default-list entry.

## Overall assessment

The suite is **well above typical WordPress-plugin maturity**: uniform 13-module architecture with zero contract deviations, a large green test suite with baseline-integrity enforcement, disciplined flag lifecycle, security-conscious throughout (centralized nonce/cap/rate-limit helper, prepared SQL everywhere, CSPRNG tokens, privacy integration), and deferrals that are labelled rather than abandoned.

What separates it from "mature product" is concentrated and tractable:
1. **One real vulnerability** (CSV formula injection) and one release-tooling bug with live consequences (theme cache-bust constant) — both same-day fixes.
2. **Performance debt in two hot paths** — the ProductSlider meta-orderby and the facet-cache's economics — plus steady background churn that a busy store pays for continuously.
3. **Single-store residue** — Hebrew literals bypassing the module's own i18n plumbing, a previous project's CSS namespace load-bearing across two modules, and one store's Elementor kit IDs inside plugin CSS — the concrete blockers to a second deployment.
4. **Duplication without a source of truth** (breakpoints ×5 files, PHP↔CSS mirrors, ignored z-index tokens) — no bugs today, all one edit away.

None of it requires schema changes or architectural rework; nearly everything above is a small, additive, individually-shippable fix.
