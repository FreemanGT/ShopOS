# B-5 audit — uninstall matrix + dead-code cross-check (2026-07-20)

The two sweeps the 2026-07-03 maturity audit deferred (B-5), run across
`shopos-core`, `shopos-digital`, `shopos-theme`. Method: 6-agent fan-out
(codegraph + grep), top findings adversarially re-verified. Each fix below is a
**separate follow-up PR** under the normal CLAUDE.md contract (flag/plan/tests/ledger);
nothing here is fixed yet.

## 1. Uninstall — created-vs-cleaned matrix

Design is sound where modules opt in: `uninstall.php` runs each module's
`on_uninstall()`, `Module_Base` sweeps that module's `shopos_core_<id>_%` options,
and Search + ShopFilters correctly drop their index tables + unschedule crons/AS.
The gaps are artifacts that fall **outside** the per-module option-prefix sweep.

### shopos-core

| Sev | Artifact | Why it leaks | Fix |
|---|---|---|---|
| **HIGH** | `{prefix}shopos_restock_subscribers` **table** (subscriber email PII) | `RestockNotify::on_uninstall()` deletes options only — never `DROP`s the table (unlike Search/ShopFilters). **Verified**: no `Database::drop()` anywhere; Module.php:170 comment even admits the PII "persists". | Add a table drop to `on_uninstall()`. ⚠ table lives in `legacy/` (Hard Rule #3 — needs owner sign-off + migration note); PII deletion is the point, so gate on owner intent. |
| MED | Recurring crons `shopos_core_product_feed_hourly`, `shopos_core_variable_stock_fix_daily_audit` | Neither module overrides `on_uninstall()` to unschedule → orphaned recurring WP-Cron rows after code removal. | `wp_clear_scheduled_hook()` in each module's `on_uninstall()`. |
| MED | Post-meta `_shopos_core_vs_sampled_color` (every variation) | VariationSwatches has no `on_uninstall`; default sweep only touches options → thousands of orphan meta rows. | `delete_post_meta_by_key()` in a new `on_uninstall`. |
| MED | Core-owned options `shopos_core_log`, `shopos_core_settings_backups`, `shopos_core_design_*` | Written by **Core classes** (not modules), so no id-prefix sweep reaches them and `uninstall.php`'s explicit 4-option delete omits them. `settings_backups` can be large; `design_*` is user config. | Add them to `uninstall.php`'s explicit delete list. |
| MED | `shopos_restock_notification_log` | Not in `OPTION_KEYS`, off the module prefix → neither sweep deletes it. (Also a dead accumulator — see §2.) | Delete on uninstall (or remove the writer per §2). |
| LOW | `shopos_core_productfeed_last_generated` | Prefix **mismatch**: key says `productfeed`, sweep matches `shopos_core_product_feed_%`. | Rename to the canonical prefix, or add to explicit delete. |
| LOW | user-meta `shopos_core_onboarding_notice_dismissed`; self-chaining `sampler_prewarm` cron; short-TTL transients | Not cleaned, but bounded / self-expiring. | Optional tidy. |
